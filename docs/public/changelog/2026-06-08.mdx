## GitHub issues can start automations

GitHub issues can now be used as the intake queue for Fabro automations. Add a `github_issue` trigger to an automation, then apply the configured label to an issue to create and start a run with the issue title, body, author, URL, repository, default branch, trigger label, and delivery ID available as workflow inputs.

```json
{
  "type": "github_issue",
  "id": "from-github-issue",
  "enabled": true,
  "trigger_label": "fabro",
  "issue_label": "Bug",
  "comment": true
}
```

Fabro links the run back to its source issue and can post a comment when the run starts or cannot start. Duplicate webhook deliveries are ignored, and a labeled issue only starts one run until the trigger label is removed and added again.
